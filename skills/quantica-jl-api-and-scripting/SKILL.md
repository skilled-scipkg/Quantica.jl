# quantica-jl: API and Scripting

## High-Signal Playbook
### Route conditions
- Use this skill for function-level API lookup, REPL docstring usage, scripting patterns, and mapping symbols to implementation files.
- Route install/backend issues to `quantica-jl-build-and-install`.
- Route physics/model-construction questions to `quantica-jl-inputs-and-modeling`.
- Route beginner recipe requests to `quantica-jl-examples-and-tutorials`.
- Route Wannier90 import specifics to `quantica-jl-src`.

### Triage questions
- Which symbol(s) are unclear (`hamiltonian`, `greenfunction`, `ldos`, `serializer`, etc.)?
- Are you in REPL exploration mode or writing scripts/modules?
- Do you need call semantics (`h(phi...)`, `g(omega)`) or indexing semantics (`h[(dn...)]`, `g[sel]`)?
- Are optional extensions involved (Makie plotting, Arpack/Arnoldi/Krylov eigensolvers)?
- Do you need performance-safe serialization/in-place workflows?

### Canonical workflow
1. Start at `docs/src/api.md` and relevant focused tutorials (`docs/src/tutorial/observables.md`, `docs/src/advanced/serializers.md`).
2. In REPL, inspect docstrings and methods: `?symbol`, `methods(symbol)`, `@which symbol(...)`.
3. Prototype a minimal script that exercises the exact symbol path.
4. If docs are ambiguous, map symbol to source via `references/source_map.md`.
5. Verify behavior with one concrete output check (shape, type, or value trend).

### Terminal quickstart (from repo root)
- `julia --project -e 'using Quantica; h = LP.square() |> onsite(4) - hopping(1) |> supercell(region = r -> norm(r) < 6); g = greenfunction(h); d = ldos(g(0.1 + 1e-3im)); @assert !isempty(d[]) && all(isfinite, d[]); println("ok:api-observables")'`

### Minimal working example
```julia
using Quantica

# REPL pattern (run interactively):
# ?hamiltonian
# methods(hamiltonian)
# @which hamiltonian(LP.honeycomb(), hopping(1))

h = LP.square() |> onsite(4) - hopping(1) |> supercell(region = r -> norm(r) < 8)
g = greenfunction(h)
d = ldos(g(0.1 + 1e-3im))
@show length(d[])
```

### Pitfalls and fixes
- `docs/src/api.md` is autogenerated index/autodocs: use symbol docstrings for argument-level details.
- Confusing call vs indexing APIs: `h(phi...)` gives Bloch matrix, `h[(dn...)]` gives harmonics.
- Extension symbols unavailable: load backend packages first (`using GLMakie`, `using Arpack`, etc.).
- Non-local conductance sign surprises: negative values can be physically expected.
- Serializer misuse: prefer `serialize!`/`deserialize!` for repeated, allocation-sensitive loops.
- Macro keyword parse errors: include `;` in macro argument lists.

### Convergence and validation checks
- `?symbol`, `methods(symbol)`, and `@which` all agree on the target code path.
- Minimal script returns expected object types/sizes.
- Serializer round trip matches original object semantics.
- Observable calls return finite outputs and expected dimensions.
- If plotting APIs are used, extension-backed methods resolve after backend import.

## Source entry points for unresolved issues
- `src/Quantica.jl`: exported public API and module include order.
- `src/docstrings.jl`: primary in-package docstring definitions for core API.
- `src/greenfunction.jl`: `greenfunction` construction, call semantics, and slicing behavior.
- `src/observables.jl`: `ldos`, `densitymatrix`, `current`, `transmission`, `conductance`, `josephson`.
- `src/serializer.jl`: serializer construction and in-place encode/decode mechanics.
- `ext/QuanticaMakieExt/docstrings.jl`: plotting API docs (`qplot`, `plotlattice`, `plotbands`, defaults).
- `ext/QuanticaArpackExt.jl`, `ext/QuanticaArnoldiMethodExt.jl`, `ext/QuanticaKrylovKitExt.jl`, `ext/QuanticaLinearMapsExt.jl`: extension method wiring for solver backends.
- Prefer targeted source search (for example: `rg -n "<symbol_or_keyword>" ext src`).
